# 卷十三 阵法与书院：多 Agent 编排

多 agent 不是人数越多越强，真正难的是秩序。若没有明责，再多代理也只是互相干扰；若没有总图，再多并行也只是群殴。龙场OS之所以借“阵法”“书院”“门派”命名，不是为了热闹，而是为了把多主体协作重新拉回责权边界。

角色先要有名，更要有职。掌门负责持总图、定方向、裁冲突，不必亲做一切，却必须掌握何者可进、何者当止。护法负责推进行动与把握节奏，是主力执行轴。长老负责经验判断与关键时刻的取舍。药王负责诊断与修复，执法负责守边界与查越权，游侠负责外探、试路、广取信息，弟子则处理大量稳定、低风险、可模板化的事务。角色之分，核心不在威风，而在让每一种行动都能找到责任主体。

多 agent 编排最常见的败局有三。其一，责任稀释，大家都参与，最后无人担责。其二，消息乱流，代理之间互相转述、重复搜证、并发写入，导致主线被碎片淹没。其三，写权限失序，谁都能改主脉，最终记忆被并行冲突撕开。要破此三病，必须把编排写成阵法：谁可以发起任务，谁可以拆解子任务，谁有否决权，谁能写气海，谁能提交经脉候选，谁有权升级到知止门或人类复核。

此卷还要把多 agent 放回控制面。真正成熟的编排，应当具备角色到权限的映射、消息总线与日志、人工介入位、冲突仲裁规则、交接协议以及失败回收机制。代理不是江湖戏服，而是制度化的职责容器。只有如此，多 agent 才不会成为热闹的技术演示，而会成为一种可长期运行的组织形态。
