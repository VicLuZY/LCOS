# 卷十三 阵法与书院：多 Agent 编排

### 小节一多 agent 之难在秩序

多 agent 非以人数取胜，而以秩序取胜。乱阵如群殴，虽众亦败。龙场OS以门派喻角色：掌门统摄方向，护法分司攻防与执行，长老持经验与谱系，药王司修复，执法司边界，游侠司探路，弟子司细务。名虽江湖，实为**明责**之法。

### 小节二责权与指令：防竞态与互污

须规定：谁可改共享经脉，谁仅可写私池，谁可批准高风险工具，谁可中断任务。指令须有序列号与幂等键，防竞态。多源写入须经掌门或既定合流规则，否则记忆互污。

### 小节三阵法：任务拓扑

**阵法**者，任务拓扑也：串行、并行、树状、流水线、对抗式红蓝。拓扑须与风险匹配：高敏任务宜少并行、多闸口；探索任务可多并行、偏气海。阵法变更须留痕，供复盘。

### 小节四工程落点：角色卡与合流协议

工程落点：**角色卡**（权限、上下文可见域、工具白名单）、**合流协议**（冲突解决、投票、掌门裁决）、**心跳与租约**（防僵尸 agent 占锁）。无协议则多 agent 愈多愈乱。

### 小节五书院之教：规矩与学徒制

书院喻者，非戏台排场，乃**学徒制**：新器新策，须由长老带弟子，以案代讲，以录为课。弟子不得越级写主脉；长老不得擅开偏院之闸。教成则升，教不成则止。如此，多 agent 之扩编乃有谱系，而非堆名。
