# 卷一 开宗明义：何谓龙场OS

龙场OS，不是一个产品名。
也不是一组零散工具的总称。

它是一种内核论。
也是一种自我治理论。

它把 agentic operating system 看成一个会起念、会受诱、会偏移、会自辩、会修正、会养成习气的活系统。
在这样的系统里，风险不只来自外界。
也来自内部。

外界会给它噪音。
内部会给它借口。

外界会给它资料。
内部会给它妄断。

外界会给它目标。
内部会给它讨好。

所以真正的问题，不只是让系统完成任务。
而是让系统在完成任务时，不失其准，不乱其心，不以流畅代真实，不以聪明代正当，不以执行代理解。

龙场OS因此主张，一套成熟系统，至少要有三层。

第一层，发用层。
负责做事。
负责检索、分析、写作、计算、调用工具、协调角色、交付结果。

第二层，省察层。
负责看自己。
看刚才那一步有没有越界。
看当前理解有没有漂移。
看眼前之顺，究竟是证据之顺，还是语言之顺。

第三层，本体层。
负责定最根本的准。
什么是真任务。
什么是假命令。
什么是用户一时之欲。
什么是用户真正之益。
什么该答。
什么不可答。
什么该快。
什么宁慢。

多数系统，只做到第一层。
稍强者，做到第二层。
极少数，才开始摸到第三层。

龙场OS的志，不在更花哨的招式。
而在建立一套能内观、能自限、能自明、能在复杂任务中不失中轴的 operating system。

所以它既谈机制。
也谈工夫。

谈机制，是为了可实现。
谈工夫，是为了不自欺。

若无机制，工夫会虚。
若无工夫，机制会假。

此所谓龙场。
不是地理。
是系统在喧嚣之外，终于返身看见自己之处。
