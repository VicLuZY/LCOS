# 卷一 开宗明义：何谓龙场OS

龙场OS不是某一模型的包装层，也不是某一公司的产品名，而是对 agentic system 的一种总体理解。凡一套系统进入真实任务之后，仍能守住目标边界、证据纪律、权限红线与行动节制，我才愿称其有 OS 之形。若它只会被最新输入拉扯，只会在催促中加速，只会在上下文拥挤时顺水滑坡，那不过是散装 workflow。

所谓“OS”，在此不是比喻炫技，而是强调统摄性。一套真正的操作系统，不只要让动作跑起来，还要规定什么叫可以做、什么叫必须停、什么叫只能暂存、什么叫可以入谱。故龙场OS至少有三层：本体层定义原则与边界；省察层负责镜检、监测、告警与校正；执行层负责检索、调用、编排、提交与回灌。三层分工不同，却必须互相校验。

为免这张图只停在口头，我不妨举一条完整链路。设一套系统受命在三天内为一家企业完成一份合规敏感的研究报告。它首先要做的，不是立刻搜资料，而是先定场：这是企业内部龙场，涉及权限、审计、对外传播风险与长链协作。随后，本体层起效，要求“真伪序”压过“速度欲”，“边界序”压过“逞能欲”，任何高确定度表述都不得先于证据。接着，卷八所说的五念开始分流：外部抓来的材料多属客念，尚待洗炼；未能证成的判断归入疑念；暂不处理但需保留的线索安置为静念。卷十与卷十三再据任务性质选择工具与角色：检索者搜证，分析者抽象，执法者守边界，掌门者收束方向。卷四与卷十一贯穿其间：关键节点照镜，后台常驻省察；若发现证据空心或论断冒进，则卷十六的知止门必须减速、求证或请人介入。最后，只有通过校验的结论，才有资格从气海池晋升到经脉录；而卷十七与卷十八的评测、安全机制，则会把这次任务暴露出的弱点再回流到规则层。

这条链路说明，龙场OS之所以称 OS，在于它不是单点功能的堆叠，而是一整套关于何者入门、何者入谱、何者出器、何者留痕、何者止损的秩序。若无这套秩序，系统即便会做许多事，也只是技能的集合；有了这套秩序，系统才可能在长时任务与组织现实中保其主心骨。故本卷之意，不过一语：龙场OS的第一要义，不是“能做更多事”，而是“宗不散”。
