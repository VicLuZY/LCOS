# 卷十八 医毒堂：诊断、安全与对抗

### 小节一医毒并建：诊伤与研毒

系统久行必有伤损，外界亦会投毒。**医毒堂**分两部：**药王谷**主诊断修复——状态漂移、记忆损坏、规则失配、错误累积、模型疲劳；**五毒堂**主对抗研究—— memory poisoning、goal hijack、prompt injection、权限诱导、长时操控。能防者众，能诊、能治、能留案者寡。

### 小节二药王：从症到方

诊断须循症：先观审计与省察之征，次查经脉与念流之污，终验工具与权限之界。方须可验证：补丁、回滚、再训练、降级、隔离。无回滚之方，慎用于生产。

### 小节三毒理：以攻为教

研毒非为炫技，乃以攻为教：自毒理反推免疫与检测特征。毒理实验须在隔离场，与主脉绝。成果入谱须双签：安全与法务（若涉数据）。

### 小节四工程落点：事件响应与 IOC

工程落点：**事件响应手册**、**IOC 清单**、**演练与复盘周期**。手册须与知止门、省察室联动，使中毒时可快速止血而非忙乱。

### 小节五合规与伦理：医毒亦须束

医毒二门，权力甚大，尤须束以合规与伦理：何类数据不可入毒理库，何类实验须额外批准，何类结论不可外泄。五毒堂所得，若反用于用户隐私之侵害，则与龙场OS之本旨相悖。故医毒堂须与场域配置表、禁脉策略一体设计，使能攻亦能守，能研亦能止。
