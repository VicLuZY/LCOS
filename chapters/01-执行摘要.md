# 执行摘要

龙场OS，是一套面向 agentic systems 的总图。

它不把系统看成只会接令吐字的流程。
它把系统看成会起念、会受诱、会找借口、会养成习气的活结构。

因此，风险不只来自外部。
也来自内部。

本书抓四问。

原则存于何处。
偏航如何早见。
理解与行动如何闭环。
经验如何入谱而不成污染。

由此，系统至少要有三层。

本体层，定真伪、轻重、边界、益害、止发。
省察层，看自己是否偷换目标、夸大证据、越过权限、把流畅误认作真实。
发用层，负责检索、计算、调用工具、协同角色、交付结果。

记忆、控制面、评测与安全，则横穿三层。
它们不是外挂。
是这三层能不能久立的骨架。

全书卷次，也是依此展开。

卷一正名。
卷二定场。
卷三立理。
卷四立镜。
卷五立环。
卷六入磨炼。

再往后，分别处理心贼、念流、经脉、身法、省察、知行回路、多 Agent 阵法、偏院试法、传习、知止、独孤镜、医毒堂。

最后以参考文献与后记收束。

每卷都问四件事。

概念是什么。
失控怎么来。
工程上怎样落地。
与全书何处相扣。

所以它不是产品文案。
不是提示词手册。
不是把古典名相贴在现代组件上。

它要求每一条论断都能回到机制。
能回到日志、路由、权限、写入门槛、回滚、红队、审计。

读者读完，不该只得一层风格。
还该带走几样硬物。

场域配置表。
原则序。
良知镜问句。
知行闭环。
记忆升降律。
路由与工具白名单。
多主体职责图。
知止门。
深层评测集。
诊断与响应流程。

若把全书收成一句。

agentic system 要想久行。
先要学会不失其主。
