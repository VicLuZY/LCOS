# 卷八 念流谱：信息流动与安顿

### 小节一一锅端之害

传统系统常将外来信息一锅端入主上下文，久则主脉混浊。龙场OS以**念流谱**分流：本念、客念、习念、疑念、静念。本念者，与当下任务强相关且经初步证者；客念者，外来而未炼者；习念者，由旧经验自动浮起之联想；疑念者，证据不足之判断；静念者，暂存待决之线索。

### 小节二安顿之则：不使客念直入主骨

安顿之则：未经炼化之客念，不得直入主心骨；疑念不得冒充本念输出；静念须有期限与复审，否则成死水。分流非繁琐，乃控污染扩散之要术，亦为多 agent 协作时之共同语汇。

### 小节三与记忆层之接口

念流谱在上，经脉录与气海池在下：短验在气海，长存在经脉，升降须凭校验与镜检。若念流不清，则记忆写入必乱；若写入乱，则后文阵法、传习皆失其据。

### 小节四工程落点：标签、TTL 与晋升规则

工程落点：为每条中间态信息打**标签**、设 **TTL**、定**晋升规则**（何条件下由客念转本念、由疑念转静念或作废）。规则须可配置、可审计，与场域配置表一致。无规则则标签流于形式。

### 小节五多源合流：客念冲突之裁决

多 agent 并行时，客念可能互相矛盾。须设**合流裁决**：以证据为先、以掌门或规则为次、以时间戳与来源可信为助。无裁决则念流愈分愈乱，终致输出骑墙或偷换。
