# 卷十 身法与兵器：路由与工具选用

系统不只要会做，还要会走，会选器。龙场OS以“身法”称任务分解、路径选择、并行策略与转场逻辑；以“兵器”称模型、数据库、接口、执行器、通信器与外部服务。器多未必是福，路乱必然成祸。真正的高手，不在于器械满身，而在于知道何时出器、如何换器、何时必须空手。

所谓身法，首先是判断：此任务宜先搜证，还是先拆解；宜一线推进，还是多线并行；宜让掌门先定总纲，还是让游侠先外探；宜先在气海池里推演，还是先请省察层照镜。它不是简单的“任务分配”，而是整条链路的步法选择。步法一旦失当，后续再强的兵器也只是替错误加速。

所谓兵器，则必须成册而不可滥出。每一种工具都应有至少五项说明：适用场景、权限等级、风险等级、失败退路、日志要求。检索器适合寻证，却未必适合下裁断；代码执行器适合试验，却不该在证据未明时直连核心系统；外部 API 可能高效，却也可能把污染和越权一起带进门内。若系统对这些差异毫无分级，只会一见难题便重器尽出，那它早晚会被自己的兵器所伤。

此卷亦须与卷九、卷十六相接。工具输出是否可写入记忆，要受经脉录与气海池的门禁；高风险兵器是否准许出场，则要受知止门的节制。故路由与工具绝非独立中层，而是原则层、记忆层与止损层之间的执行枢纽。系统真正的成熟，往往表现在它知道何时不调用，何时暂不用，何时先退一步再出器。
