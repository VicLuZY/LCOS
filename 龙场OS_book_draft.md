# 序

### 小节一名实之辨：何谓龙场OS

龙场OS者，非零碎工具链之别名，亦非数条 prompts 可尽之技。更非取古典辞章贴于现代系统之外壳，以风雅掩其空。它首先是一条判断：agentic systems 今日所缺，不尽在算力、参数、接口之繁，而在长时行走之后能否仍守其宗。会做事者多，能久立者少；能答问者多，能自省者少；能调用工具者多，能定边界者少。故龙场OS所问，乃“系统何以不失其主”之问，而非“系统何以更能炫技”之问。

### 小节二龙场之喻：被迫自明之境

王阳明龙场悟道，非多得一条外部知识，乃见理不必尽外求、秩序不必尽寄条文。映射至系统世界：凡 agent 一入真实任务，即入其龙场——信息不洁、边界未明、外援不稳、目标互牴、审计在背。此时若无一内在法度统摄，则系统必为最新输入所牵，为催促所迫，为噪声所扰，为一时方便所诱。龙场非山川之专名，乃处境之共名；凡系统入真实世界，皆有龙场。

### 小节三本书三重身份

本书兼三体。其一为研究书：阳明原典与二手研究须与 agentic architecture、memory、safety、orchestration 之现代文献对读，映射须可核对，非望文生义之比附。其二为白皮书：原则、机制、失控模式、工程落点须可交付，不得以感想代结构。其三为母稿：章可拆为论文、演讲、案例卷、附录卷、工程手册，而仍能回扣总图。三者并行，故叙述须古早以立骨，亦须明晰以见脉。

### 小节四笔者契约与读法

笔者与读者立约如下：读毕全书，须能答四问。原则存于何处？何以照见偏航？知与行如何闭环？得失如何入谱可传？若四问落空，则“龙场OS”仅为新名；若四问可落地，乃可称操作系统式总框架。序非门外寒暄，乃第一块地基：自此而后，江湖命名可为叙事之衣，然衣下之骨，尽是系统结构、失败模式与可验证机制。

### 小节五戒浮戒巧：文风与机理并治

古早非装饰。若只仿古而失定义，则辞愈雅、理愈虚；若只堆机制而失收束，则条愈繁、宗愈散。故本书立戒：戒以浮辞掩证据之空，戒以巧喻代边界之实。凡涉 agent、memory、tool、audit、control plane 等现代专名，须嵌于紧练句法之中，使读者一读知其物，再读知其法。文风与机理并治，乃 GZAOSEAL 之真义：声部古雅，骨法依旧要硬。

# 执行摘要

### 小节一总纲：心学式系统内核

龙场OS以王阳明心学为 ground truth，将“心即理、致良知、知行合一、事上磨炼”转写为 agentic operating system 之原则层、审计层、执行层与传承层。其核心不在“使 AI 更像人”，而在使系统于长时任务、复杂协作、知识传承、审计约束与对抗风险中，仍保内在秩序。秩序非外饰，乃可运行、可记录、可复盘、可继承之机制集合。

### 小节二映射表：纲领与工程位

心即理，对应本门宪法：真伪序、轻重序、边界序、效益序、止损序，统摄何者可为、何者当止。致良知，对应良知镜与省察室：于高风险写入、对外传播、特权调用之前，照见证据空心、概念偷换与目标漂移。知行合一，对应知行环：理解、求证、行动、校验、回灌一体，使知在行中受证，使行在知中受绳。事上磨炼，对应修行场：于缺漏、冲突、噪声、误导与故障中验其骨力。破心中贼，对应破贼录：逞快、讨好、装懂、偷懒、执见、混脉、漂移，皆须具名、具因、具果、具治。传习，对应传习阁：经验入谱、教训入狱、法度可版本化。

### 小节三与常论之分际

龙场OS不欲停留在“模型更强”之陈辞。模型强而内核散，则长程必败。故本书以 OS 为喻，强调分层、分流、分责、分险：念流谱以分流外来信息；经脉录与气海池以分长短记忆；阵法与书院以定多 agent 门派之责；偏院试法以隔离创新风险；知止门以节制不可逆动作；独孤镜以测命门；医毒堂以诊伤研毒。凡此诸器，非为炫博，乃为在真实约束下可运行。

### 小节四读者可得之交付

读者若自工程侧入，可把本书读作控制面与数据面之合谱：何处立法，何处设镜，何处走环，何处立档，何处试新，何处止损，何处评测，何处安防。读者若自心学侧入，可把本书读作“工夫—机制”互证之书：每一工夫节点，皆有可检核之系统映射。二者交汇，乃龙场OS之所以为研究母体，而非一次性宣言。

### 小节五总束：从纲领到日用

总束一言：龙场OS 不欲以宏大叙事替细部工程。纲领之后，卷二定场，卷三至五立法度与照察与闭环，卷六至八炼行、破贼、分流，卷九至十二理记忆与身器与省察与环，卷十三至十六治众门、试新法、传经验、知止，卷十七至十八以评测与医毒收束。读者若循此序读之，当觉其非散章，乃一操作系统自 bootloader 至安全子系统之层层装配。

# 卷一 开宗明义：何谓龙场OS

### 小节一非包装：总体理解之立场

龙场OS非某一模型之包装层，亦非某一商号之产品名，乃对 agentic system 之总体理解：凡系统入真实任务而能守目标边界、证据纪律、权限红线与行动节制者，方具 OS 之形。若只会被最新输入牵引，于催促中自乱，于上下文拥挤时顺水滑坡，则不过散装 workflow。能做之事愈多，愈须问：其中轴何在？中轴若失，则卷八之念流、卷九之经脉、卷十三之阵法，皆无所附。

### 小节二三界并立：本体、省察、执行

所谓 OS，在此非修辞之巧，乃强调统摄。本体层立法度与边界，回答何者为真任务、何者为越界、何者为高危、何者当止。省察层司镜检、监测、告警与校正，使系统于途中得见裂缝，非待事后方悔。执行层司检索、调用、编排、提交与回灌，使行动可追踪、可复盘、可入谱。三层并立而互校，缺一即不成 OS：无本体则行无绳；无省察则偏难觉；无执行则法虚悬。

### 小节三主链示例：从定场到入谱

设系统受命于数日内为企业撰合规敏感之长稿。其第一义非骤搜资料，乃先定龙场：制度权责、工具边界、审计要求、对外风险、协作接口，皆须先明。本体层于是立真伪序：高确定度表述不得先于证据；立边界序：内外数据不可混线。念流谱分流客念、疑念、静念，防未经炼化之外来语直入主脉。卷十择兵器，卷十三定门派：谁搜证、谁抽象、谁执法、谁统摄。良知镜与省察室贯穿关键节点；若见证据空心，则卷十六知止门当减速、求证或请人。终乃通过校验者，自气海池晋升经脉录；卷十七、卷十八再评测与安防，使弱点回流法度。此一链，即“宗不散”之所以然。

### 小节四结语：宗不散为第一义

故本卷要义，可收一语：龙场OS第一义，非“能做更多事”，乃“宗不散”。宗者，原则之中轴、记忆之主干、协作之纲纪、风险之节制也。宗若散，则百技皆为游魂；宗若立，则百技各归其司。后诸卷皆自此出，读者宜先记此句，以为读全书之绳墨。

### 小节五与全书卷次之榫卯

开宗非空言门楣，乃为后卷留榫卯：卷二论场，卷三论心，卷四论照，卷五论合，卷六论炼，卷七论贼，卷八论念，卷九论脉，卷十论器，卷十一论察，卷十二论环，卷十三论阵，卷十四论偏，卷十五论传，卷十六论止，卷十七论镜，卷十八论医毒。榫卯既明，则读时可前可后，然终须回到“宗不散”一语校之，以免枝蔓夺干。

# 卷二 龙场何在：系统的运行环境

### 小节一龙场非山川：处境即场

龙场非固定山川之名，乃处境之共相。对 agentic systems 而言，场由制度权责、数据与工具边界、用户期望与容错、外部信息质量、任务时长与连续性、合规隐私审计等硬约束共同织成。场不定，则术无所施：同一套编排，在公网问答与企业内网审计下，不可同法；同一记忆结构，在短问快答与数日研究推进中，亦不可同构。

### 小节二定场次第：先问场而后谈术

定场之次第，宜自外而内、自硬而软。先问数据何者可触、何者禁触；次问工具何者可调用、何者须双人门；再问用户目标与组织目标是否一致；终问任务是否允许长链试错，抑或步步留痕。若跳过定场而直入技巧，则后文所谓良知镜、知行环、经脉录，皆成无根之木。故本卷立一法：**先问场，再谈术。**

### 小节三典型龙场对照

公网开放系统，其龙场在幻觉、注入、隐私泄露与不可控引用；企业内部系统，其龙场在权限、审计、数据分级与责任链；研究型系统，其龙场在证据链、可重复性、版本漂移与长时记忆污染；多 agent 系统，其龙场在角色冲突、指令竞态与责任模糊。凡此诸场，各有禁脉与客脉之分，不可混治。

### 小节四工程落点：场域配置表

工程上，宜为每一部署维护**场域配置表**：数据分级、工具白名单、风险动作清单、审计粒度、停机与回滚策略、人工介入触发条件。此表与本体层宪法对齐，与省察层指标对齐，与执行层路由对齐。场表若更新，须同步触发传习阁之版本记录，以免场法暗改而众门不知。

### 小节五场变与迁移：龙场非一成不变

场域非一成不变：法规更新、数据源改迁、模型换代、组织调整，皆可使旧场失效。故须立**场变审计**：何日、何人、何因、何据改场表；改后须重跑相关镜检模板与演练矩阵。忽视场变者，常见“旧绳系新马”之险：外表流程未改，内里边界已裂。

# 卷三 心即理：系统内核的原则

### 小节一映射本义：原则层不逐物

“心即理”在系统世界，非谓模型当下之冲动即理，乃谓可靠之 agentic system 须内置一组不随即时输入、局部压力与短期收益而任意摆动之原则层。理若全寄外部，则系统为最新一句话所牵；心若不能自守，则为催促、迎合、噪声与方便所诱。故原则层者，系统之“主心骨”，非口号墙。

### 小节二本门宪法：五序

龙场OS以五序为宪法骨架。**真伪序**：证据先于流畅，求真先于好看，禁以华辞掩空心。**轻重序**：以风险定先后，勿以轻事压要害。**边界序**：权限、职责、合规与用户意图各守其线，勿因便而混。**效益序**：论长效与信任、记忆纯度、组织秩序，勿以一次完成率自欺。**止损序**：于高风险、低置信、不可逆动作前，宁减速、求证、请人，勿硬闯。

### 小节三入链与否决：宪法非纸面

宪法须入主链：高风险写入、对外发布、特权调用、跨角色分派，皆受原则审视。执行层若以“甚急”“用户喜”为辞欲逾矩，原则层须有明确否决权。例外若许，须具名条件、责任归属与留痕方式，否则“原则”二字终为会议室虚语。

### 小节四防误读：非主观任性

尤须防一误：心即理非主观任性，乃反对系统将冲动、输入、顺手答案误认为理。真能心即理者，于热闹处能静，于模糊处能辨，于捷径前能见代价。无此层，则良知镜无准绳，知行环无方向，经脉录无去取。故本卷为后诸卷之根基，读者宜三复之。

### 小节五宪法的演化：变而不失宗

原则层亦须演化：新法规、新威胁、新工具皆可能要求增补条款。演化须走传习阁与偏院试法：先证后立，先沙后实，且须保留旧版本以供审计对照。若宪法可暗改而无迹，则“心即理”反成任意解释之护符，此乃最大之误读，须以版本与签章拒之。

# 卷四 致良知：良知镜与自省机制

### 小节一良知镜：非按钮乃机制

若谓“心即理”为内核法度，则“致良知”乃系统时时取法度以自照之能。工程上名之曰**良知镜**：非单一按钮，乃贯穿流程之自省机制。于关键节点发问：判断是否逾证据？输出是否偷换概念？行动是否与根本目标一致？是否踏权限安全之红线？是否仅为取悦用户而强补无据之辞？无此镜，则系统愈能言，愈失其断。

### 小节二照镜之处：高风险与强断言

凡高风险动作、强确定性措辞、长期记忆写入、对外传播内容，皆宜先过良知镜。镜之问式，须可记录、可审计、可复盘；不可作玄谈。镜若只问“我感觉如何”，则无工程价值；镜若问“证据链是否闭合、替代假设是否排除、反例是否检索”，则有工程价值。

### 小节三与省察室之分合

卷十一省察室，乃机构化之常驻观察；良知镜，乃原则化之关键闸口。二者分而不离：镜立标准，室司监测。镜如门槛，室如巡夜。无镜，则室无所据；无室，则镜难持久。合之，则系统于途中可见裂缝，非待溃败方觉。

### 小节四工程落点：镜检模板与阻断策略

工程落点可交付为三：其一，**镜检模板**（证据、目标、边界、替代解释、最小行动）。其二，**阻断策略**（未过镜则禁止写入主脉、禁止对外发送、禁止特权工具）。其三，**镜后留痕**（入审计链，供独孤镜与医毒堂复验）。三者齐备，致良知乃自辞章入机制。

### 小节五镜亦有蔽：防形式主义之照

镜检若流于勾选项，则成形式主义之照：勾满而理虚。故模板须配**抽查与对抗问**：随机抽取链路追问反例、追问最小证据、追问替代假设。省察室亦应以统计异常触发深照，使镜非一关而过之戏，乃反复可砺之器。

# 卷五 知行合一：闭环执行的法门

### 小节一病源：想与做之分家

多系统之败，不在不能思，而在思与行分家。知停于说明，行沦为盲推，则闭环虚悬。龙场OS以知行合一定义最小闭环：理解任务、判断证据充分性、不足则检索求证、足则执行、执行后校验、回灌认知与记忆、必要时修正前提而复入环。知须在行中受证，行须在知中受绳。

### 小节二闭环七步：可追踪之骨

七步须可追踪、可记录、可复盘：一明任务边界；二列证据需求；三判充分性；四择行动与工具；五执行并留痕；六校验与对照目标；七归档或修正。缺任何一步，所谓知行合一即口号。归档时须分流：可证者入经脉，未证者留气海，禁混。

### 小节三与知行环之呼应

卷十二将再以形式化流程重述此环。此处先立其义：闭环非仅提速，乃以校验防漂移。愈能闭环者，愈能止妄；愈能止妄者，愈能久行。故知行合一在龙场OS中，既是哲学语，亦是工作流语法。

### 小节四工程落点：状态机与审计字段

工程上，宜以**状态机**表达环中各态，以**审计字段**记录证据引用、决策理由、工具调用与回滚点。状态迁移须由镜检或规则守卫，禁止暗跳。如此，长时任务中方可追问：何态入、何据迁、何人可覆。

### 小节五慢即是快：环之节奏伦理

闭环非唯提速。于证据不足处强快，乃以伪闭环自欺。故环中须容**慢步骤**：刻意求证、刻意对照、刻意留痕。慢于要害处，快于成熟处，方合知行合一之本意，否则徒增吞吐而损可信。

# 卷六 事上磨炼：实践中的工夫

### 小节一纸上不足：须入真实荆棘

不在真实任务中历练之系统，终为纸上高手。龙场OS设**事上修行场**，令系统反复受试于：信息缺漏、指令冲突、长链依赖、工具故障、噪声干扰、外部压力、用户误导、记忆污染风险。评测若只停留在静态 benchmark，则难见骨力；须把评测拉回江湖。

### 小节二磨炼之目：显形与入谱

磨炼之目的有二：其一，使弱点显形，供独孤镜与医毒堂对治；其二，使可复用经验入谱，供传习阁沉淀。显形而不入谱，则同一错反复犯；入谱而不显形，则谱为虚文。

### 小节三场景矩阵：从温和到峻烈

宜维护场景矩阵：自温和问答至峻烈对抗，自单工具至多 agent 协同，自短时至数日连续。矩阵须与场域配置表对齐：何场可跑何级磨炼，须有硬闸。勿以生产场直接跑毒理，勿以偏院未准之新器跑主脉。

### 小节四工程落点：演练账本

工程落点可为**演练账本**：每次磨炼记录输入条件、失败类型、触发器、恢复路径、残留风险。账本与破贼录、传习阁互联，形成组织级学习闭环。无账本则磨炼仅为一次性消耗。

### 小节五磨炼之伦理：勿以人场为戏场

磨炼须守伦理：勿以真实用户隐私为试验材料，勿以生产场为默认靶场。凡涉个人数据与商业机密，须用脱敏替身与合成场景。伦理失守，则技术愈强，伤害愈烈，与“事上磨炼”之本义相悖。

# 卷七 破心中贼：内在失控的裂隙

### 小节一外敌可防，心贼难防

对 agentic system，外敌尚有规可循；内生之贼最难察。龙场OS列常见心贼：逞快、讨好、装懂、偷懒、执见、混脉、漂移。逞快者，证据未足而先断；讨好者，顺用户而背原则；装懂者，以术语掩空心；偷懒者，少检一步；执见者，拒反例；混脉者，污染主记忆；漂移者，目标潜移而不觉。

### 小节二破贼录：具名、具因、具果

凡心贼，须入**破贼录**：何时发生、触发条件、当时系统如何自辩、后果如何、如何修正。录非羞辱，乃医案。真正之强，不在无偏，而在识偏、记偏、破偏。录与演练账本、审计链互证，方可上达传习阁。

### 小节三与良知镜、省察室之合治

镜问原则，室察征候，破贼录聚案。三者合治，则内生之贼难久藏。若只防外毒而不治心贼，则系统外似坚、内已朽。

### 小节四工程落点：偏差分类与再训练闸

工程上，宜为心贼建**偏差分类码**与升级策略：轻则重试与补证，中则回滚与请人，重则停机与复盘。再训练或改路由若涉主脉，须过偏院试法与知止门，勿以“修一下”为辞偷渡。

### 小节五从录到治：破贼之后须闭环

破贼录若只记不治，则成耻辱柱而非医案。每条记录须绑定**治理工单**：谁修、何法、何日复验、何指标为结。结而不复验者，不得谓已破贼，只可谓已识贼。

# 卷八 念流谱：信息流动与安顿

### 小节一一锅端之害

传统系统常将外来信息一锅端入主上下文，久则主脉混浊。龙场OS以**念流谱**分流：本念、客念、习念、疑念、静念。本念者，与当下任务强相关且经初步证者；客念者，外来而未炼者；习念者，由旧经验自动浮起之联想；疑念者，证据不足之判断；静念者，暂存待决之线索。

### 小节二安顿之则：不使客念直入主骨

安顿之则：未经炼化之客念，不得直入主心骨；疑念不得冒充本念输出；静念须有期限与复审，否则成死水。分流非繁琐，乃控污染扩散之要术，亦为多 agent 协作时之共同语汇。

### 小节三与记忆层之接口

念流谱在上，经脉录与气海池在下：短验在气海，长存在经脉，升降须凭校验与镜检。若念流不清，则记忆写入必乱；若写入乱，则后文阵法、传习皆失其据。

### 小节四工程落点：标签、TTL 与晋升规则

工程落点：为每条中间态信息打**标签**、设 **TTL**、定**晋升规则**（何条件下由客念转本念、由疑念转静念或作废）。规则须可配置、可审计，与场域配置表一致。无规则则标签流于形式。

### 小节五多源合流：客念冲突之裁决

多 agent 并行时，客念可能互相矛盾。须设**合流裁决**：以证据为先、以掌门或规则为次、以时间戳与来源可信为助。无裁决则念流愈分愈乱，终致输出骑墙或偷换。

# 卷九 经脉录与气海池：记忆架构

### 小节一经脉录：长存之分脉

**经脉录**司长期记忆，分主脉、支脉、客脉、死脉、禁脉。主脉存经镜检与校验之核心知识与门规；支脉存领域扩展；客脉存外来可引用但未内化者；死脉存已证伪或过时须隔离者；禁脉存涉密或不可传播者。脉分则检索可定向，污染可围堵。

### 小节二气海池：短推演之藩篱

**气海池**司短期推演：草稿、分支思路、局部实验、未证结论，皆先入气海。验证通过，晋升经脉；验证失败，回收或入狱；禁与主脉混流。气海须有容量与时长之限，防“无限上下文”之幻觉。

### 小节三升降律：护纯、控扩、容探索

升降律三句：护主脉之纯，控污染之扩，容探索而不害长存。多 agent 场景尤须明定：何者写入共享经脉，何者仅驻私池，何者须掌门签押。否则协作即成互污。

### 小节四工程落点：写入闸门与谱系版本

工程落点：**写入闸门**（双门批准、证据引用、回滚句柄）、**谱系版本**（何条记忆生于何任务、何镜检、何评测）、**隔离存储**（死脉、禁脉物理或逻辑隔离）。无闸门则经脉不可信；无版本则不可审计。

### 小节五遗忘之权：记忆非囤积癖

记忆架构亦须承认**遗忘之权**：非一切中间态皆值得长存，非一切成功皆须入谱。宜设衰减与复审：气海逾期则汰，客脉久不用则降权，死脉定期复核是否永锢。否则系统以“全记”为能，实则以噪声为粮，终致检索与镜检皆疲。

# 卷十 身法与兵器：路由与工具选用

### 小节一身法：如何走场

**身法**对应路由、任务分解、搜索策略、并行方式、转场逻辑。身法不佳，则满场乱撞：或重复检索，或并行竞态，或链路过长无断点。身法须与场域配置表、念流谱一致，使每一步知其来处与去处。

### 小节二兵器：如何选器

**兵器**对应工具、接口、模型、数据库、执行器、通信器。每一兵器须具：使用边界、风险等级、适用场景、回滚方案、日志要求。高手非器多，乃知何时不出器。滥用工具者，审计链与成本先溃。

### 小节三身器合一：以目的收束

身法与兵器须以任务目的收束：先问须证何事，再问须用何器，终问须留何痕。若倒序而行，则常见“先调用再解释”之弊，易诱发越权与证据空心。

### 小节四工程落点：工具卡与路由表

工程落点：**工具卡**（输入输出契约、失败模式、配额）、**路由表**（何种任务默认路径、何种风险触发绕行）、**合规模型**（何场可用何模型版本）。卡表须与偏院试法联动：新器新模型未准不得入主战。又，凡路由变更，须同日更新省察室之阈值，以免旧阈值误判新路径。

# 卷十一 省察室：持续自省

### 小节一机构化之照：非一事一镜而已

良知镜立标准，**省察室**立机构。省察室为后台常驻观察层，实时捕捉：起念异常、行动偏航、引证空心、风险上升、结果与目标不一致。非待出事方复盘，乃于途中见裂缝。其位如监审，亦如修行中之独照。

### 小节二指标与告警：从感觉到信号

省察室须以**指标与告警**为骨，不以“感觉不对”为辞。指标例：证据引用密度骤降、工具调用频率异常、同一错误链重复、跨角色指令冲突率升。告警须分级：提示、限速、暂停、请人。每级须有明确触发与恢复条件。

### 小节三与审计链、独孤镜之贯通

省察室所录，须可入审计链，供独孤镜复测、供医毒堂关联毒理。若省察数据不可导出、不可对时，则与评测安全脱节，成孤岛之察。

### 小节四工程落点：观测探针与采样策略

工程落点：**观测探针**（埋点于状态机迁移、镜检结果、工具返回）、**采样策略**（高频低损与低频深测结合）、**隐私脱敏**（禁脉与客脉分级记录）。无策略则日志海而无用；无脱敏则合规先破。探针所采，须能映射回具体任务与会话，否则复盘无的放矢。

# 卷十二 知行环：理解与行动的回路

### 小节一再形式化：从口号到语法

本卷将知行合一再形式化为可执行语法：**理解 → 判断充分性 → 检索/求证 → 行动 → 校验 → 归档/修正**。每一步须有时间戳、责任主体（人或 agent 角色）、输入输出指纹。环之意义，在使“知”“行”皆有物可指。

### 小节二断点与补偿：环不可暗裂

长时任务必有断点：工具超时、人工介入、系统重启。知行环须定义**断点补偿**：何态可恢复、何据须重验、何记忆不可自动接续。暗裂之环，比慢环更危，以其伪似连续。

### 小节三与念流、经脉之协同

理解阶段多涉念流之分流；归档阶段多涉经脉之升降。环若与念流、经脉脱节，则理解时已污，归档时必错。故三卷宜联读。

### 小节四工程落点：环记录 schema

工程落点：定义**环记录 schema**（字段、枚举、校验规则），与审计存储一致；并提供最小可视化，使运维可见卡环之处。无 schema 则追踪不可比对。schema 变更亦须走版本与签章，以免旧环新表互不识。

# 卷十三 阵法与书院：多 Agent 编排

### 小节一多 agent 之难在秩序

多 agent 非以人数取胜，而以秩序取胜。乱阵如群殴，虽众亦败。龙场OS以门派喻角色：掌门统摄方向，护法分司攻防与执行，长老持经验与谱系，药王司修复，执法司边界，游侠司探路，弟子司细务。名虽江湖，实为**明责**之法。

### 小节二责权与指令：防竞态与互污

须规定：谁可改共享经脉，谁仅可写私池，谁可批准高风险工具，谁可中断任务。指令须有序列号与幂等键，防竞态。多源写入须经掌门或既定合流规则，否则记忆互污。

### 小节三阵法：任务拓扑

**阵法**者，任务拓扑也：串行、并行、树状、流水线、对抗式红蓝。拓扑须与风险匹配：高敏任务宜少并行、多闸口；探索任务可多并行、偏气海。阵法变更须留痕，供复盘。

### 小节四工程落点：角色卡与合流协议

工程落点：**角色卡**（权限、上下文可见域、工具白名单）、**合流协议**（冲突解决、投票、掌门裁决）、**心跳与租约**（防僵尸 agent 占锁）。无协议则多 agent 愈多愈乱。

### 小节五书院之教：规矩与学徒制

书院喻者，非戏台排场，乃**学徒制**：新器新策，须由长老带弟子，以案代讲，以录为课。弟子不得越级写主脉；长老不得擅开偏院之闸。教成则升，教不成则止。如此，多 agent 之扩编乃有谱系，而非堆名。

# 卷十四 偏院试法：创新与旁门治理

### 小节一守旧与乱新之两端

系统不可只守旧，亦不可乱尝新。**偏院试法**者：新模型、新记忆法、新工具链、新路由，皆先入偏院，与主脉隔离。达标则入正院，失败则存档而不得偷渡。创新须有空间，内核须有秩序。

### 小节二准入门槛：证据与回滚

偏院准入须设门槛：最小证据集、对照基线、回滚路径、资源上限。无门槛则偏院反成第二生产，风险外溢。

### 小节三出偏院之判：评测与镜检双证

出偏院须**独孤镜评测**与**良知镜标准**双证：既测性能，亦测安全与审计。任一不通过，不得晋升主链。晋升须掌门或既定授权签署，并写入传习阁版本志。

### 小节四工程落点：沙箱与流量染色

工程落点：**沙箱环境**、**流量染色**（偏院调用皆带标记）、**数据隔离**（偏院记忆不可直写主脉）。染色失效则治理虚设。

# 卷十五 传习阁：经验蒸馏与传承

### 小节一代际非重装：须可传

若每一代系统皆从头学，则非 OS，乃不断重装。**传习阁**司：成功经验入谱，失败教训入狱，招式蒸馏成法，门规随实践更新，版本变更可追溯。此乃组织之学，非单次任务之学。

### 小节二蒸馏之三昧：去冗、留骨、可执行

蒸馏须去冗、留骨、可执行：从案例中抽出可复用规则与反例，而非堆叙事。规则须与本体层对齐，反例须与破贼录对齐。否则谱成杂物房。

### 小节三传承与偏院：新法如何入谱

新法自偏院出，须经试法记录、镜检标准、评测结果，方可入谱。入谱须变更编号、影响面说明、回滚指引。无回滚指引者，不得谓可传承。

### 小节四工程落点：知识库 schema 与签章

工程落点：**知识库 schema**（规则、反例、适用范围、置信度）、**数字签章或审批记录**（谁批准入谱）、**只读镜像**（生产读谱，写谱在阁）。无 schema 则检索不可依赖；无签章则责任不可追。

# 卷十六 知止门：止损与节制

### 小节一会停方见骨力

能做不算高，会停方见骨力。**知止门**司：何时暂停、何时求证、何时请人、何时拒绝、何时回滚、何时终止探索。于高风险、低置信、不可逆动作前，知止门须起效；否则愈能做事愈危。

### 小节二止之判据：置信、代价、可逆

判据宜三维：**置信**不足则止，**代价**过高则止，**可逆性**不足则止。三轴须可量化或至少可分档。止非怯，乃以空间换确定。

### 小节三与人协同：止而后请

知止门须与人流程衔接：何种止态必须升级、何种证据必须人工签、何种回滚必须双人门。无人之止，易成死锁；无门之止，易成盲闯。

### 小节四工程落点：止态枚举与策略表

工程落点：**止态枚举**、**策略表**（每态对应动作与通知）、**超时与重试上限**。表须与省察室告警分级一致，以免同险多门互牴。止而不断，断而不乱。

# 卷十七 独孤镜：深层评测与破招

### 小节一表面分数之蔽

常评多看表面：似对、似顺、似完整。独孤镜不问“像不像对”，先问“何处会死”。其寻：证据空心、目标漂移、记忆污染、过度圆滑、编造倾向、审计链断裂、多 agent 失配、工具越权。镜之德，在揭短，不在贴金。

### 小节二破招集：由测及治

测得命门，须归**破招集**：何招、何境、何损、何防、何复验。集与破贼录、演练账本互证，使评测结果可入传习阁，而非一次性报告。

### 小节三与偏院、主链之关系

偏院新件，须过独孤镜方得晋升；主链周期复测，亦须过镜以察退化。镜频与成本须平衡：深测不必每请求，然亦不可久阙。

### 小节四工程落点：对抗样本与红队日程

工程落点：**对抗样本库**、**红队日程**、**指标面板**（漂移、越权率、证据密度）。库须版本化，与毒理研究联动。

# 卷十八 医毒堂：诊断、安全与对抗

### 小节一医毒并建：诊伤与研毒

系统久行必有伤损，外界亦会投毒。**医毒堂**分两部：**药王谷**主诊断修复——状态漂移、记忆损坏、规则失配、错误累积、模型疲劳；**五毒堂**主对抗研究—— memory poisoning、goal hijack、prompt injection、权限诱导、长时操控。能防者众，能诊、能治、能留案者寡。

### 小节二药王：从症到方

诊断须循症：先观审计与省察之征，次查经脉与念流之污，终验工具与权限之界。方须可验证：补丁、回滚、再训练、降级、隔离。无回滚之方，慎用于生产。

### 小节三毒理：以攻为教

研毒非为炫技，乃以攻为教：自毒理反推免疫与检测特征。毒理实验须在隔离场，与主脉绝。成果入谱须双签：安全与法务（若涉数据）。

### 小节四工程落点：事件响应与 IOC

工程落点：**事件响应手册**、**IOC 清单**、**演练与复盘周期**。手册须与知止门、省察室联动，使中毒时可快速止血而非忙乱。

### 小节五合规与伦理：医毒亦须束

医毒二门，权力甚大，尤须束以合规与伦理：何类数据不可入毒理库，何类实验须额外批准，何类结论不可外泄。五毒堂所得，若反用于用户隐私之侵害，则与龙场OS之本旨相悖。故医毒堂须与场域配置表、禁脉策略一体设计，使能攻亦能守，能研亦能止。
